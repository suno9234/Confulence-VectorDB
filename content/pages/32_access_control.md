# 접근 권한 관리

## 원칙: 최소 권한 (Least Privilege)

필요한 리소스에 필요한 수준의 권한만 부여합니다.
"일단 다 줘도 되지 않나?"는 넥서스AI의 보안 원칙에 위배됩니다.

## 권한 등급

| 등급 | 설명 | 예시 대상 |
|------|------|-----------|
| Viewer | 읽기 전용 | 인턴, 타 부서 참관 |
| Developer | 읽기 + 코드 Push | 개발직 일반 |
| Senior | Developer + PR 승인 | 시니어 개발자 |
| Admin | 시스템 설정 변경 가능 | 팀장, 인프라팀 |
| Super Admin | 전체 권한 | CTO, IT지원팀장 |

## 시스템별 권한 구성

### GitLab
- `Reporter`: 코드 읽기, 이슈 생성
- `Developer`: 브랜치 push, MR 생성
- `Maintainer`: main 브랜치 머지, 설정 변경
- `Owner`: 레포 삭제 등 최고 권한

### AWS
- 개발자: `ReadOnlyAccess` + `S3FullAccess` (개발 버킷만)
- 인프라팀: `PowerUserAccess`
- 인프라팀장/CTO: `AdministratorAccess`
- IAM 사용자 직접 생성 금지 (SSO 연동)

### 데이터베이스 (운영)
- 일반 개발자: 운영 DB 접근 불가
- 시니어 이상: 읽기 전용 계정 (팀장 요청 필요)
- DBA(인프라팀): 전체 권한

## 권한 신청 절차

1. 슬랙 #it-support 채널에 요청
   ```
   [권한 신청]
   - 시스템: GitLab
   - 레포: nexusbot-api
   - 요청 권한: Developer
   - 사유: 신규 입사로 개발 업무 시작
   ```
2. 직속 팀장 승인 (슬랙 이모지 ✅ 또는 댓글)
3. IT지원팀 또는 인프라팀 처리 (1영업일 이내)
4. 권한 부여 완료 알림

## 권한 회수 기준

| 사유 | 처리 기간 |
|------|-----------|
| 퇴직 | 퇴직 당일 전 계정 비활성화 |
| 부서 이동 | 이동 후 5영업일 내 기존 권한 회수 |
| 장기 휴직 (30일 이상) | 휴직 시작일 |
| 보안 위반 | 즉시 |

## 분기별 권한 감사

- 분기마다 IT지원팀이 전 시스템 권한 목록 추출
- 팀장들이 팀원 권한 적정성 검토 및 승인/회수 처리
- 결과는 `보안 감사 > 접근권한 감사 > YYYY-QX` 폴더에 보관
