# 비밀번호 정책

최종 개정: 2025년 1월 2일

## 비밀번호 요구 사항

| 항목 | 기준 |
|------|------|
| 최소 길이 | 12자 이상 |
| 대문자 | 1개 이상 포함 |
| 소문자 | 1개 이상 포함 |
| 숫자 | 1개 이상 포함 |
| 특수문자 | 1개 이상 포함 (!@#$%^&*) |
| 변경 주기 | 90일마다 강제 변경 |
| 이전 비밀번호 재사용 | 최근 5개 재사용 금지 |
| 연속 실패 잠금 | 5회 실패 시 15분 잠금 |

## 시스템별 비밀번호 정책 적용 현황

| 시스템 | 비밀번호 정책 | MFA |
|--------|--------------|-----|
| Google Workspace | 적용 (Google 정책) | 필수 (Google Authenticator) |
| GitLab | 적용 | 필수 |
| Jira / Confluence | SSO 연동 (Google) | Google MFA 적용됨 |
| Jenkins | 적용 | 필수 |
| AWS Console | IAM 정책 적용 | 필수 (TOTP) |
| Vault | 적용 | 필수 |
| SSH | 비밀번호 로그인 불가 (키 인증만) | — |

## MFA 설정 방법

### Google Authenticator (권장)
1. 앱스토어에서 "Google Authenticator" 설치
2. 구글 계정 → 보안 → 2단계 인증 → 앱 설정
3. QR 코드 스캔

### 백업 코드
- 첫 MFA 설정 시 제공되는 백업 코드 반드시 안전한 곳에 보관
- 백업 코드 분실 시: IT지원팀에 신원 확인 후 재설정 요청

## 비밀번호 관리 도구

- **권장 도구**: 1Password (회사 계정 제공, 팀별 보관함 운영)
- **1Password 계정 발급**: IT지원팀 이도현에게 요청
- **금지**: 브라우저 자동 저장, 메모장/엑셀에 비밀번호 기록

## 비밀번호 유출 시 대응

1. 즉시 해당 시스템 비밀번호 변경
2. security@nexusai.co.kr 에 신고 (어떤 계정이 언제 유출 의심되는지)
3. IT지원팀과 함께 접속 로그 확인
4. 필요 시 해당 계정 일시 잠금 조치
